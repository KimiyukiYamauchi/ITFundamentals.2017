### CPU => 演算装置/制御装置/内部レジスタ

#### 性能
- クロック周波数 => １秒あたりの同期の回数(Hz)
    - 1000 => 1k
    - 1000k => 1M(メガ)
    - 1000M => 1G(ギガ)
    - 1000G => 1T(テラ)
    - 0.001 => 1m
    - 0.001m => 1μ(マイクロ)
    - 0.001μ => 1n(ナノ)
    - 0.001n =>1p(ピコ)
- コア数 => 演算や制御を実行するための「中核」の回路の数
  - マルチコアプロセッサ/シングルコアプロセッサ
- 32/64bit => プロセッサが一度に扱える情報量
- CPUキャッシュ => CPUとメインメモリの間のバッファをおこなう高速なメモリ
- ディスクキャッシュ => メインメモリとハードディスクの間のバッファを行うメモリエリア

#### プロセッサの種類

### GPU(Graphics Processing Unit)
  - 画像処理を行う半導体チップ、CPUの負荷を軽減

#### VRAM

### メモリの種類
  - ROM => 不揮発性 => 電気の供給がなくてもデータを保持する
  - RAM => 揮発性 => 電気の供給がないとデータが消えてしまう
    - SRAM => 高速/高価、キャッシュメモリ、リフレッシュの必要がない
    - DRAM => 低速/比較的安価、メインメモリ、リフレッシュ必要

### メモリモジュール
- デスクトップPC => DIMM
- ノート型PC => S.O.DIMM
- 切り欠き => メモリモジュールを間違った方向に指すのを防ぐ

### メモリの性能
- SDRAM -> DDR SDRAM -> DDR2 SDRAM -> DDR3 SDRAM
- 種類の異なるDRAMには互換性がなく、マザーボードのメモリスロットの形状を確認
- 増設時、現在刺さっているモジュールと同一規格、同一メーカーのものを使用する

### ハードディスク vs SSD(ソリッドステートドライブ)

#### ハードディスクの問題点
- 衝撃や落下に弱い
- 消費電力が大きく、低電力化が難しい
- 振動が発生し、音がうるさい
- 読み書きの高速化が難しい

#### SSDの問題点
- ハードディスクの問題点は解決するが高価

### ハードディスクの種類
- 接続方法 => 内蔵、外付け

## 有線接続
### ネットワーク機器
- ネットワクカード(NIC) => MACアドレス(物理アドレス) => 6バイト(前半3バイト:ベンダーID、後半3バイトがボードごとの番号)
- ハブ(スイッチングハブ(データリンク層)/リピータハブ(物理層))
- スイッチ(データリンク層) => 効率的にデータを送る仕組みがある
- ルータ(ネットワーク層) => 異なるネットワーク同士を接続する

### ケーブル
- 同軸ケーブル => 1本の動線の周囲をプラスチックの被膜および絶縁カバーで保護
- ツイストペアケーブル => 8本の銅線2本ずつよれら4組の束ねられている
- 光ファイバーケーブル => 光信号で通信、ケーブルの距離が長くなっても信号が減衰しない
- ケーブルのコネクタ => RJ-45(8pin)
- 電話で使用される => RJ-11(6pin)
- ストレートケーブルとクロスオーバーケーブル
    - ストレートケーブル => PCとスイッチの接続、スイッチとルータ
    - クロスオーバーケーブル => PCとPCの接続、スイッチとスイッチ
    - オートMDI/MDIX => 接続先のポートを自動的に判断する
- xx Base y => xx => 通信速度、y => 2(200)、5(500)
- xx Base -Tx => xx => 通信速度、-T => ツイストペアケーブルを使う
- xx Base-Fx => xx => 通信速度、-Fx => 光ファイバケーブル
- ツイストペアケーブルのカテゴリ
    - cat3 => 10Mbps => 10Base-T
    - cat5 => 100Mbps => 100Base-Tx
    -cat5e,cat6 => 1000Mbps => 1000Base-T,1000Base-Tx

## 無線接続
### IEEE802.11 => 無線の規格
- a => 5GHz、54Mbps
- b => 2.4GHz、11Mbps
- g => 2.4GHz、54Mbps
- n => 2.4GHz/5GHz、600Mbps

### 無線LANの機器

    - 暗号化 => WEP/WPA/WPA2


### 無線LAN接続の携帯

### その他の無線通信技術

### プロトコル
- ネットワーク内でのコンピュータ同士が通信をする時の手順や約束事

### TCP/IP => プロトコルスタック
1. アプリケーション層 => HTTP, TELNET, SNTP, POP3, SMTP, FTP, DHCP, NFS
1. トランスポート層 => TCP/UDP、通信の信頼線を確保、ポート番号
1. インターネット層 => IP(ホスト間のパケットのやり取りを実現), ICMP(ping)、ARP(IPアドレス -> MACアドレス)、IPアドレス
1. ネットワークインタフェース層 =>  イーサネット、IEEE802.11、Token Ring、MACアドレス

### IPアドレス
#### IPv4
- 32ビット(4バイト)
- 8ビットずつ4つの組に分け、それぞれの組みをピリオドで区切って10進表記
- ネットワーク部、ホスト部 => サブネットマスク
- デフォルトゲートウェイ => ルータのIPアドレス
- DNSサーバ(ドメイン名 <-> IPアドレス)

#### TCP/IPの主なプロトコル
- SMTP => 電子メールを送信する
- POP3/IMAP => 電子メールを受信する(サーバーのメールボックスからメールを取り込む)
- HTTP(80) => Webページ
- DNS => IPアドレスドメイン名の変換
- DHCP => IPアドレスを自動的に割り振る

##### セキュア通信プロトコル
- SSL/TLS => 通信相手の認証や通信の暗号化、内容の改ざん検出を可能とする
- SSH(22) => セキュアなリモート接続
- HTTPS(443) => セキュアなHTTP

### ネットワーク共有

#### クライアント/サーバー
コンピュータを、サービスを提供するサーバとサービスを利用するクライアントに分け、役割分担をして運用するしくみ

#### ピアツーピア
- クライアント/サーバの役割を切り替える
    - ローカルアドホックネットワーク => 無線LAN、Bluetooth機器同士の接続
    - ダイレクトリンク => USB転送ツールケーブル、有線LAN、アクセスポイントを使用した無線LANで接続
    - オンラインピアツーピアネットワーク => 大規模なファイル共有(winny、bittorrent)、オンラインゲームサービス

#### Windowsネットワーク
- ドメイン => ネットワーク全体を管理するドメインサーバが配置されている
- ワークグループ => 小規模なネットワーク、一元管理するサーバなし
- ホームグループ => ごく小規模なネットワークで、共有を容易に行うことができる

#### データ共有デバイス

### ネットワークプリンターとローカルプリンター
- ローカルプリンター => PCに説ぞ則されているプリンタ
- ネットワークプリンター => LANに接続できるプリンタ

### クラウドコンピューティング
ネットワーク上(インターネット)のサーバが提供するサービスを、ユーザがそのサーバを意識することなく、必要に応じて必要な文だけ利用できる
- オフサイト => 離れた場所でサービスを提供する
- オンプレミス => 組織の構内にサーバを設置・導入して、自前で運用する形態

#### クラウドを利用するメリット
- 迅速なシステム導入
- ネットワークさえつながればアクセスできる
- リソースのカスタマイズができる
- トータルコストの削減
- 管理者負担の軽減

#### サービスの提供形態
- SaaS => app + midle ware + os + h/w => アプリケーションのサービスを提供、gmail
- PaaS => middle ware + os + h/w => アプリケーションプログラムだけに注力できる、google app engine、HEROKU
- IaaS => os + h/w => PaaSより自由度が高い、Amazon EC2


#### 仮想化
- サーバーの仮想化 => 従来までのサーバ構築は、1台の物理サーバに対して1つのサーバーOSを導入するのが一般的。しかし、近年ではサーバ仮想化により、物理サーバーのもつハードウェアリソースを共有し、組織内やクラウド環境でのリソースを効率的に活用する。仮想化を実現するためのソフトウェアである仮想化ソフトウェアの導入により、容易に実装。

### セキュリティとは
- 情報資産 => 組織が保有する情報にかかわるすべての資産
- 脅威 => 情報資産を脅かす外的要因
- 機密性 => 盗聴されない、暗号化
- 完全性 => 勝手に書き換えられないようにする、認証
- 可用性 => 使えたい時に使える

### 情報セキュリティマネジメントシステム(ISMS)
- PDCAアプローチ
    - Plan(計画) => 情報セキュリティポリシーを策定し、目的や適用範囲を定義
    - Do(実施) => 情報セキュリティ対策を講じてシステムの運用を行う。
    - Check(評価) => 情報セキュリティ水準を評価する
    - Act(改善) => 情報セキュリティの見直しを行い改善する。

- 情報セキュリティポリシー =>セキュリティの基本方針を文書化し、組織内全員に周知徹底

- ユーザトレーニング => 人的脅威に対する対策

### 脅威の種類
- マルウェア
    - ウィルス => 自己伝染機能/潜伏機能/発病機能
    - ワーム => 自己増殖を繰り返しながら不正動作を行う
    - トロイの木馬 => 無害なソフトウェアを装いながらシステムやコンピュータに侵入、寄生や自己増殖はなし
    - スパイウェア => ユーザに関する情報を勝手に送信する
    - アドウェア => ポップアップなどによる広告表示
    - ランサムウェア => 金銭やデータを求めてユーザを脅迫する

- フィッシング => Webサイトやメールをなりすまし、個人情報を盗む詐欺行為
- スパム => 迷惑メール
- パスワードクラック => 辞書攻撃/総当り攻撃(ブルートフォースアタック)
- ソーシャルエンジニアリング => 「社会的な(social)」手段により、重要な情報を不正に収集する

### 基本的なセキュリティ対策
- アンチマルウェアソフト(アンチウイルスソフト)
    - パターンファイル(定義ファイル)による照合によるマルウェアを検出、駆除。
    - 常に最新の定義ファイルに更新しておく => 自動更新の設定
- ユーザ認証
    - 知っていること => パスワード
    - 持っているもの => ICカード
    - その人自身 => 生体認証
    - 他要素認証/二要素認証
- 暗号化 => データを決まった規則にしたがって変換する
    - アルゴリズム => 変換のルール
    - 鍵 => 変換に必要な情報
    - 暗号化 <-> 復号
- 利用していない機能の無効化
    - Bluetooth
    - NFC
    - Guestアカウント
- 公共のコンピューターを利用するリスク
- ファイルとプリンターの共有

### パスワードマネジメント
- パスワード管理
    - デフォルトのパスワードを変更
    - パスワードの複雑性
    - 機密性の高いパスワード
    - パスワードの有効期限
    - パスワードの再利用
    - シングルサインオン => 一度の認証で複数のサービスを利用できるようにするしくみ、Kerberos

### ネットワークのセキュリティ
- 無線LAN
    - 管理者パスワード => APのデフォルトパスワードを変更
    - SSIDの設定
    - 認証と暗号化
        - WEP => 暗号化のみ、脆弱性が指摘されているので使わないほうがいい
        - WPA => 暗号化 -> TKIP、認証 -> エンタープライズモ-ド、パーソナルモード
        - WPA2 => 暗号化 -> AES(CCMP, WPA2-AES)、認証 -> エンタープライズモ-ド、パーソナルモード
    - 暗号キーの変更
    - SSIDブロードキャスト
        - 電波が届く範囲の無線LANクライアントに向けて、SSIDをブロードキャストする
        - 無効にすることによって、セキュリティを高めることができる
    - MACアドレスフィルタリング
        - 特定のMACアドレスからのみ接続を許可する機能
        - MACアドレスをなりすまして不正アクセス
    - ファームウェアのアップデート
    - オープンWi-FiとセキュアWi-Fi
- インターネットアクセスのセキュリティ
    - ファイアウォール => 内部ネットワークと外部ネットワークの境界で、不正アクセスをフィルタリングする
        - ソフトウェアファイアウォール
        - パーソナルファイアウォール
    - パケットフィルタリング
        - アクセス制御リスト(ACL) => MACアドレス、IPアドレス、ポート番号

### 安全なWebサイトの閲覧方法
- ブラウザの設定
    - 不必要な/疑わしいブラウザプラグイン、ツールバー、拡張機能の無効化
        - プラグイン => ブラウザの機能を拡張するプログラム
    - フォーム/パスワードの自動入力(オートコンプリート)の無効化
    - キャッシュ、履歴、Cookieの設定
        - キャッシュ => 過去にアクセスしたWebページを一時的にコンピュータ内に保存
        - 履歴 => 訪問したWebサイトの履歴
        - Cookie => Webサイト(Webサーバー)がWebサイトの訪問者のコンピューター(クライアント)にWebブラウザを通じて書き込むファイル。
    - 信頼できないソースであるという警告を確認する
    - Webサイトのブロック
    - 公共のコンピュータでのインターネット利用
- Webサイトの安全性の確認
    - セキュアな接続とWebサイトであることを確認
        - URLが「https://」となっている
        - 鍵マークの表示
    - 無効な証明証である警告がでないか確認する
    - 疑わしいリンクでないか確認する
    - 疑わしいバナー広告でないか確認する
    - アドウェアによる症状が出ていないか確認する
    - 個人情報(PII)の利用制限
        - ユーザ名
        - パスワード
        - PIN(暗証番号)
        - 銀行口座番号
        - クレジットカード番号
        - 母親の旧姓

### 物理的セキュリティ
- 物理的なセキュリティ対策
    - ハードウェアの盗難防止
        - 耐タンパ性
    - ソフトウェア/ライセンスの盗難防止
    - ショルダーサーフィンの防止
        - ショルダーサーフィン => 肩越しに画面や利用者の手の動きからパスワードなどを盗み見る行為
    - ダンプスターダイビングの防止
        - ダンプスターダイビング => ごみあさり、ユーザーが捨てたごみからパスワードや講座情報などを入手する行為
- デバイスのハードニング
    - ハードニング => 脆弱性を減らす
    - 画面のロック
    - ハードウェアの暗号化
